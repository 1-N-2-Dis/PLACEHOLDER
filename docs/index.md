## 0. Source-of-truth map (one fact, one home)

To stop drift and contradictions, **each concern has exactly one canonical owner.** Every other
doc *links* to it — it never restates it. **When two docs disagree, the canonical owner wins**
until the loser is reconciled.

| Concern | Canonical owner | Note |
|---------|-----------------|------|
| Vision · problem · who it's for | [idea.md](../idea.md) | the brief this suite was generated from |
| What we build (features `F-###`, journeys `UJ-###`, business rules `BR-###`) | [PRD](03-prd.md) | stable feature IDs; origin of the spine |
| How it's built (architecture, components, **map/routing stack**, trade-offs) | [System Design](06-system-design.md) | owns the MapLibre/OpenFreeMap + ORS decision |
| Data schema · entities · Firestore rules shape | [Data Model](09-data-model.md) | — |
| Tests · traceability (every `F-###` → ≥1 test) | [QA Test Plan](11-qa-test-plan.md) | the traceability home |
| Security · auth/authz · compliance · secrets | [Security & Compliance](12-security-compliance.md) | network surfaces are exposed → present |
| Evidence · research leads (`said/did/paid`, tiered) | [Evidence Register](evidence-register.md) | desk research + seed-pin hypotheses; not validation |
| Significant decisions (the *why*) | [ADRs](adr/) | append-only; supersede, never edit. Map stack = ADR-0001 |

**The rule:** a fact lives in its canonical doc; everything else links to it. Do not restate a
fact owned elsewhere — restating is how two copies drift and the next contradiction is born.
Example: the map/routing stack is owned by **System Design**. The PRD, Data Model, QA, and
Security docs mention it in one line and link here — they do **not** describe it. Change it in
one place (System Design + an ADR) and nothing else can contradict.
