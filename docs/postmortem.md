## Proposal protocol after run closure

The Framework Observer may add an `FC-###` proposal only after inspecting run evidence, the final
implementation plan, final docs, tests, demo result, and available user or judge feedback. Each
proposal must name its source event IDs, owner classification, observed friction, counterevidence,
baseline, target, observation window, and failure/rollback condition.

The product lead—not the observer—chooses whether to trial, adopt, revise, reject, or revert a
proposal. Framework templates and playbooks are never edited autonomously from this document.
