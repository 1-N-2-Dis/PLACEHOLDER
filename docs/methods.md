# Methods — GuidHer routing and freshness ledger

> **Scope:** code-grounded derived values only. This document does not create a place safety
> score or a crime rating. Routing penalties are operational route-selection inputs, and report
> severity remains a per-report signal under INV-005.

## Confidence rule

The route engine’s constants and graph are checked-in implementation inputs, so their values are
high-confidence code facts. Their real-world safety effect is unvalidated; route safety language
must remain directional (for example, “safest we found”), not a precision safety claim.

## Equation registry

| ID | Output | Method | Inputs | Confidence |
|---|---|---|---|---|
| EQ-001 | Edge traversal cost | `edge_length × highway_multiplier × severity_multiplier` | DS-001, DS-002 | High for code behavior; low for real-world safety effect |
| EQ-002 | Active report status | `now - (lastActivityAt or createdAt) ≤ 24 hours` | DS-003 | High for code behavior |
| EQ-003 | Routes returned | penalized A* route plus unpenalized shortest route when their edge paths differ; maximum two | DS-001, DS-002 | High for code behavior |

For EQ-001, the highway multiplier is `1.0` for footway class and `1.15` otherwise. A red report
within 90 m applies `×1000`; a yellow report within 60 m applies `×5`; green applies no avoidance
penalty. These are documented code constants, not risk probabilities or public place scores.

## Dataset / input registry

| ID | Input | Source | Confidence |
|---|---|---|---|
| DS-001 | Preprocessed PUP pedestrian graph | `frontend/public/graph/pup-20km.bin`; generated by `scripts/build-graph.mjs` | High for available graph artifact |
| DS-002 | Fresh red/yellow report locations and per-report severity | Supabase `reports` rows normalized by `frontend/src/lib/reports.js` and passed through `frontend/src/lib/routing.js` | Medium: user-generated inputs and demo-mode AI behavior |
| DS-003 | `createdAt` / `lastActivityAt` and 24-hour window | `frontend/src/lib/freshness.js`, mirrored server-side | High for implementation behavior |

## Traceability

- F-003 and F-005 use EQ-001 through EQ-003.
- QA must test freshness boundaries, red-unavoidable routing, and the two-route cap.
- No UI surface may convert these calculations into a neighborhood rating, incident count claim,
  or guarantee of safety.
